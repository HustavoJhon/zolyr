Planteamiento del Problema

Contexto Bancario

  • Problema: Atención lenta y sin priorización en sucursales bancarias.
  • Causas:
    • Tiempos de espera prolongados.
    • Falta de prioridad para clientes VIP.
    • No existe trazabilidad de transacciones.
    • Errores manuales difíciles de revertir.

Objetivo del Proyecto

Desarrollar un sistema de simulación bancaria que optimice la atención al cliente aplicando estructuras de datos dinámicas (Lista, Cola y Pila) para registrar, priorizar y revertir operaciones.

Estructuras de Datos y Operaciones

1️⃣ Lista Enlazada — Registro Maestro de Clientes

  • Función: Almacenar datos personales.
  • Operaciones: insertar(), eliminar(), buscar(), listar().
  • Ventaja: Inserciones rápidas sin redimensionar memoria.

2️⃣ Cola de Prioridad — Gestión de Atención

  • Función: Organizar turnos por tipo de cliente.
  • Prioridades: VIP → Preferencial → Regular.
  • Operaciones: encolar(), desencolar(), consultarFrente().

3️⃣ Pila — Registro de Transacciones

  • Función: Controlar el historial de operaciones.
  • Operaciones: push(), pop(), peek().
  • Ventaja: Permite la función undo para revertir acciones.

Algoritmos Principales

Pseudocódigo — AgregarCliente (Lista Enlazada)

ALGORITMO AgregarCliente(dni, nombre, tipo)
    nuevo ← CREAR Nodo
    nuevo.dni ← dni
    nuevo.nombre ← nombre
    nuevo.tipo ← tipo
    nuevo.siguiente ← NULL
    
    SI cabeza = NULL ENTONCES
        cabeza ← nuevo
    SINO
        actual ← cabeza
        MIENTRAS actual.siguiente ≠ NULL HACER
            actual ← actual.siguiente
        FIN MIENTRAS
        actual.siguiente ← nuevo
    FIN SI
    
    MOSTRAR "Cliente agregado exitosamente"
FIN ALGORITMO

Algoritmos Principales

Pseudocódigo — RegistrarTransaccion (Pila)

ALGORITMO RegistrarTransaccion(dniCliente, tipoOperacion, monto)
    transaccion ← CREAR Nodo
    transaccion.dni ← dniCliente
    transaccion.tipo ← tipoOperacion
    transaccion.monto ← monto
    transaccion.fecha ← ObtenerFechaHora()
    
    transaccion.siguiente ← tope
    tope ← transaccion
    
    MOSTRAR "Transacción registrada"
FIN ALGORITMO

Apéndice

Ejemplo en C++ — Inserción en Lista Enlazada

struct Cliente {
    string dni, nombre, tipo;
    Cliente* siguiente;
    Cliente(string d, string n, string t)
        : dni(d), nombre(n), tipo(t), siguiente(nullptr) {}
};

void agregarCliente(Cliente*& cabeza, string dni, string nombre, string tipo) {
    Cliente* nuevo = new Cliente(dni, nombre, tipo);
    if (!cabeza) cabeza = nuevo;
    else {
        Cliente* actual = cabeza;
        while (actual->siguiente) actual = actual->siguiente;
        actual->siguiente = nuevo;
    }
    cout << "Cliente registrado exitosamente.\n";
}
