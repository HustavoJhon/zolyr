# Carpeta tasks/

La carpeta `tasks/` contiene la documentacion relacionada a la gestion de tareas del proyecto FINPROC. Su objetivo es registrar y organizar el trabajo del equipo usando un enfoque simple compatible con SCRUM y con los plugins de Obsidian como Tasks, Kanban y Dataview.

Esta carpeta no contiene codigo fuente. Solo incluye archivos de organizacion y control del avance del proyecto.

## Contenido esperado

### 1. roles.md
Archivo que describe los roles del equipo. Por ejemplo:
- Scrum Master
- Analista
- Desarrollador
- Documentador
- Tester

El objetivo es dejar claro quien es responsable de cada parte del proyecto.

### 2. schedule.md
Archivo con el cronograma de tareas por sprint. Puede incluir:
- fechas de inicio y fin de cada sprint
- tareas planificadas
- responsables
- dependencias
- hitos del proyecto

### 3. backlog.md (opcional)
Lista general de tareas pendientes del producto. Aqui se pueden registrar todas las funciones pedidos en el informe o requerimientos del curso.

### 4. sprint_tasks.md (opcional)
Listado de tareas organizadas por sprint. Este archivo se usa junto con el plugin Obsidian Tasks.

Ejemplo simple:

```md
- [ ] Implementar lista enlazada
- [ ] Implementar cola de prioridad
- [ ] Implementar pila de transacciones
- [ ] Crear modulo de persistencia
- [ ] Integrar estructuras al menu principal
```

### 5. integracion_con_obsidian.md (opcional)

Explica como usar los plugins:

- Tasks
- Kanban
- Dataview
- Agile Task Notes
    

para gestionar el proyecto dentro del vault.

## Notas importantes

- Esta carpeta almacena organizacion, no codigo.
- Se recomienda mantener un formato simple y claro por cada sprint.
- No colocar archivos compilados ni codigo fuente aqui.
- Los archivos deben reflejar solo gestion del trabajo.
